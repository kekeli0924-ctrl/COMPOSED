# Composed

A personal soccer training companion that helps youth players (U12-U18) log sessions, track progress, get AI-powered video analysis, and follow structured training programs. Built for players, coaches, and parents.

## What It Does

**For Players:**
- Log training sessions with detailed stats (shooting, passing, fitness, RPE, body check)
- Follow guided live sessions with drill-by-drill timers and audio cues
- Record training video with rear camera while running a guided session (dual-mode)
- Upload video for AI analysis (Gemini) that detects shots, passes, and technique
- Client-side video compression (FFmpeg.wasm) reduces upload size by ~10x
- Track progress with IDP goals (Technical, Tactical, Physical, Psychological)
- Earn XP, level up, and unlock 15+ badges
- View personal records, streaks, weekly reports, and trend charts
- Browse 76 drills with unique SVG diagrams and position-specific filtering
- Enroll in 4 structured training programs (Finishing Mastery, Ball Mastery, Complete Player, Speed & Agility)

**For Coaches:**
- Manage a roster of players via invite codes
- Assign training plans to specific players on specific dates
- View player stats, compliance rates, and development progress

**For Parents:**
- Read-only dashboard showing child's training activity, streaks, goals, and coach feedback
- Connect via invite code generated by the player
- Player-controlled privacy settings (hide ratings, coach feedback, or IDP goals)

## Tech Stack

- **Frontend:** React 19, Vite, Tailwind CSS v4, Recharts, react-webcam
- **Backend:** Express 5, better-sqlite3, JWT authentication
- **AI:** Google Gemini (video analysis), Anthropic Claude (AI chat)
- **Video:** FFmpeg.wasm (client-side compression), MediaRecorder (session recording), tus (resumable upload)
- **PWA:** Offline support, installable, service worker caching

## Getting Started

### Prerequisites
- Node.js 20+
- npm 9+

### Setup

```bash
git clone https://github.com/kekeli0924-ctrl/COMPOSED.git
cd COMPOSED
npm install
```

### Environment Variables

Create a `.env` file in the root:

```env
# Required for AI features (get free key at https://ai.google.dev)
GEMINI_API_KEY=your-gemini-api-key

# Required in production
JWT_SECRET=your-secure-random-secret

# Optional
CORS_ORIGIN=http://localhost:5173
PORT=3001
```

### Run Development

```bash
npm run dev
```

This starts both the Vite frontend (port 5173) and the Express backend (port 3001) concurrently.

### Build for Production

```bash
npm run build
npm start
```

## Key Features in Detail

### Dual-Mode Session Recording
Players prop their phone up (rear camera facing the field) and run a guided session. The screen shows drill timers and controls overlaid on the live camera feed. When the session ends, the recorded video is ready for AI analysis. NoSleep.js keeps the screen awake during 30-60 minute outdoor sessions.

### AI Video Analysis
Videos are compressed client-side (720p, H.264, ~10x smaller) before upload. The compressed video is sent to Google Gemini which detects shots, passes, foot usage, and estimates RPE. Results auto-fill the session log.

### Authentication
JWT-based auth with 30-day access tokens and 90-day refresh tokens. Three roles: player, coach, parent. All data is scoped by user_id — each user only sees their own sessions, plans, and goals.

### Training Programs
Four preset programs with week-by-week periodized sessions. Players enroll in one active program at a time, progressing through daily sessions across multiple weeks.

## Database

SQLite with 22 tables and 18 migrations. Key tables: sessions, users, settings, training_plans, idp_goals, video_analyses, drills, programs, parent_player_links.

## API

50+ REST endpoints organized by feature. All protected by JWT in production. Rate limited (300 req/15min general, 20 req/15min auth).
